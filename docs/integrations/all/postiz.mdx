## Setup guide

### Step 1: Create a Postiz OAuth app

In Postiz, go to **Settings > Developers > Apps** and create an OAuth application.

Copy:

- **Client ID** (starts with `pca_`)
- **Client Secret** (starts with `pcs_`)

Set your redirect URL to the one Nango shows during integration setup.

### Step 2: Configure the integration in Nango

Use these credentials in Nango:

- **Client ID**
- **Client Secret**

### Step 3: Authorize the connection

Nango will redirect users to Postiz consent at:

- `https://platform.postiz.com/oauth/authorize`

After approval, Nango exchanges the returned code at:

- `https://api.postiz.com/oauth/token`

## API gotchas

- Postiz public API tokens are sent in the `Authorization` header without a `Bearer ` prefix.
- Postiz OAuth access tokens currently do not expire according to the public docs.
- Public API base URL is `https://api.postiz.com/public/v1` for Postiz Cloud.
